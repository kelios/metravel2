# UI-гайд Metravel (основные правила)

Документ фиксирует базовые правила UI, чтобы интерфейс был **ровным**, **нейтральным**, легко поддерживаемым и единым на web + mobile.

## 1) Общий принцип

- Один продукт — один стиль.
- Предсказуемая верстка важнее декоративности.
- Любая “яркость” должна быть функциональной (ошибка/успех/предупреждение), а не “для красоты”.

## 1.1) Процесс разработки (обязательное правило)

- **Перед каждым шагом разработки** (любая правка/задача) сначала ознакомься с релевантной документацией в `docs/`.
  - Примеры:
    - для главной страницы: `docs/home-page-improvements.md`
    - для страницы путешествия: `docs/travel-details-redesign.md`
    - для карты: `docs/map-page-architecture.md`
- **После каждого логического шага** (не в конце дня, а именно после шага) обязательно запускай:
  - `npm run lint`
  - `npm run test`

## 2) Изображения и плейсхолдеры (если изображения нет)

### 2.1 Когда показываем плейсхолдер
- Если `imageUrl` отсутствует.
- Если загрузка изображения упала (onError).

### 2.2 Правило “нейтральной заглушки”
Плейсхолдер должен быть **нейтральным**:
- без иконок
- без текста “нет изображения”
- без эмодзи
- без ярких акцентных цветов

### 2.3 Геометрия и стабильность
- Плейсхолдер обязан сохранять **те же размеры/радиусы**, что и реальный медиа-блок.
- Нельзя допускать “прыжков” карточек из-за смены высоты/контента.

### 2.4 Визуальный стиль плейсхолдера
- Фон: `DESIGN_TOKENS.colors.backgroundSecondary`
- Бордер: `DESIGN_TOKENS.colors.borderLight` (тонкий, 1px)
- Радиус: как у контейнера (обычно `DESIGN_TOKENS.radii.lg`)

## 3) Иконки

### 3.1 Запрещено
- Разноцветные “случайные” иконки в интерфейсе.
- Эмодзи в продакшен UI (в текстах, заголовках, подсказках, кнопках).

### 3.2 Разрешённый стиль
Мы используем иконки в **двух согласованных стилях**:
- **Серо‑зелёный (primary)** — основной нейтральный стиль.
- **Оранжевый (accent)** — только для акцентов (например, CTA/выделение), но дозировано.

Источник иконок должен быть единым (например, `MaterialIcons`/`react-native-vector-icons`), а цвет берётся из дизайн‑токенов.

Рекомендуемые цвета:
- Основной: `DESIGN_TOKENS.colors.primary`
- Мьют: `DESIGN_TOKENS.colors.textMuted`

## 4) Цвета и статусы

- Ошибка: только через токены (например, `DESIGN_TOKENS.colors.danger`).
- Успех/предупреждение — тоже только через токены.
- Не добавляем новые “случайные” hex-цвета в компоненты без необходимости.

## 5) Текст, подписи и микрокопирайт

- Текст должен быть коротким и функциональным.
- Не пишем в карточках “Нет изображения” — это шум.
- В подсказках и статусах не используем эмодзи.

## 6) Где править

- Общие цвета/радиусы: `constants/designSystem` (`DESIGN_TOKENS`).
- Карточки путешествий: компоненты в `components/travel/*` и `components/listTravel/*`.
- Карта/фильтры: `components/MapPage/*`.

---

## Чек-лист перед мерджем

- Нет эмодзи в UI.
- Нет ярких разноцветных иконок.
- Плейсхолдеры нейтральные и не ломают сетку.
- Цвета/радиусы берутся из `DESIGN_TOKENS`.
