# metravel-icon-art

Регламент генерации одной растровой иконки/арт-ассета (или набора в едином стиле)
через AI-картинку. Источник схемы промптов и палитры — `docs/ICON_ART_PROMPTS.md`
(палитра — `constants/modernMattePalette.ts`). Цель — ассет в фирменном матовом
стиле MeTravel (тёплая оранжевая доминанта `#f5842c`, мягкий свет, без глянца/неона).

Сама модель-генератор в репо не зашита (как и у `metravel-badge`): скилл собирает
**production-ready промпт**, размеры и пайплайн постобработки; картинку отдаёт во
внешнюю image-модель (запускает владелец / бэк), затем размещает результат в проекте.

## Граница ответственности

- `app.json` — в do-not-touch (AGENTS.md). Пути app-иконок/splash/favicon **не менять
  без явного запроса**; нужная правка — предлагать диффом, не применять молча.
- Растр в фичевых компонентах — только через `components/ui/ImageCardMedia.tsx`
  (прямой `expo-image` запрещён ESLint-гвардом). Travel-карточки — через `UnifiedTravelCard`.
- Если ассет = **одноцветный плоский глиф** — предложить SVG-компонент (как
  `BelarusOutlineIcon`/`NavigationIcon`), а не PNG: легче, чётче, темизируется. Растр —
  для многоцветных/объёмных/иллюстративных ассетов.
- Бэкенд не правим. Если ассет должен лежать в БД/S3 (как badge) — тикет на BE через
  агент `ticket-board` (`area=back`), а не локальный файл.

## Принципы

1. **Один стиль на набор.** Выбери ОДИН `STYLE_PRESET` (flat-line / duotone / soft-3d /
   engraving / illustration) и держи его на всю серию + один `seed`-прогон, иначе ассеты
   не выглядят семьёй.
2. **Одна доминанта.** Обычно `brand #f5842c`; 1–2 поддерживающих цвета только из
   `modernMattePalette.ts`. Без цветов вне палитры, без неона/кислоты.
3. **Не дублировать.** Сверь существующие ассеты в `assets/icons/`, `assets/images/`,
   `assets/travel/`, `assets/quests/` — не плодить близнецов.
4. **Прозрачность чистая.** PNG с alpha без полупрозрачного ореола (иначе на dark-теме
   и в круглой рамке вылезает фон).
5. **Размер от UI.** Генерим крупно (`≥1024²` app/hero, `≥512²` иконки), даунскейлим в
   нужные `@1x/@2x/@3x`.

## Шаги

### 1. Спека ассета
Определи: **тип** (UI-глиф / app-иконка / контент-арт — раздел 7 промпт-доки),
назначение, где используется (компонент/экран/слот), нужные размеры и аспект (1:1 / 4:3 /
16:9), прозрачный фон или нет, тёмный вариант нужен ли. Один ассет = один `SUBJECT`.

### 2. AI-промпт
Собери строго по схеме `docs/ICON_ART_PROMPTS.md`:
`BASE_STYLE, STYLE_PRESET, PALETTE, SUBJECT. FRAMING. NEGATIVE`.
- `STYLE_PRESET` — по типу ассета (раздел 3 доки).
- `PALETTE` — доминанта `#f5842c` + поддержка из палитры (раздел 4).
- `SUBJECT` — конкретика иконки.
- `FRAMING` — дефолт (раздел 5) с оверрайдом по типу (раздел 7); `NEGATIVE` — раздел 6.
Положи готовый промпт рядом с ассетом (коммит-сообщение / тикет) для перегенерации.
Для набора — собери промпты пачкой с общими BASE_STYLE/STYLE_PRESET/seed.

### 3. Генерация
Картинку рендерит внешняя image-модель (запуск — владелец/бэк; модель в репо не зашита).
Если в окружении доступен image-gen MCP/инструмент — использовать его; иначе отдать промпт
владельцу и принять готовый PNG. Не подставлять фейковые URL/плейсхолдеры в код.

### 4. Постобработка
Pure-Node, без внешних деп (образец — `scripts/generate-monochrome-icon.js`, zlib/raw PNG):
автокроп до объекта, выбивание фона до alpha (если модель добавила фон), даунскейл в
`@1x/@2x/@3x`, монохром-вариант для Android adaptive. Проверить чистоту alpha по краям.

### 5. Размещение
Положи по таблице раздела 9 промпт-доки:
- UI-глиф → `assets/icons/<группа>/name.png` (+ `@2x`,`@3x`).
- App-набор → `assets/images/` (`icon.png`, `adaptive-icon.png`, `monochrome-icon.png`),
  `assets/icons/logo_*`. ⚠️ Правка `app.json` — только по явному запросу, диффом.
- Контент-арт → `assets/travel/` · `assets/quests/<id>/` · `assets/images/<кампания>/`.
`assetBundlePatterns` уже покрывает эти корни — новые подпапки бандлятся сами.

### 6. Подключение в код
- Контент/маркетинг-картинка в карточке/hero → через `ImageCardMedia` (fit=contain + blur,
  фото-доминанта по AGENTS.md). Не `cover`, не прямой `expo-image`.
- UI-глиф → `require('@/assets/icons/...')` в Image-обёртке проекта; для одноцветного —
  предпочтительно SVG-компонент.

### 7. Верификация
- `npm run check:image-architecture` (использование изображений) и `npm run check:fast`.
- Браузер: открой экран с ассетом (preview), проверь рендер light+dark, мелкий и крупный
  размер, чистоту краёв; для app-иконки — adaptive-маска (центр в безопасной зоне).
- Сам проверяешь в браузере/на устройстве — не перекладывать проверку на пользователя.

## Что не делать

- Не менять `app.json`/пути иконок без явного запроса — только диффом.
- Не вставлять прямой `expo-image` в фичевые компоненты; не обходить `ImageCardMedia`.
- Не подставлять фейковые URL/плейсхолдеры; не коммитить ассет без реальной картинки.
- Не смешивать STYLE_PRESET в одном наборе; не вводить цвета вне `modernMattePalette.ts`.
- Не генерить растр там, где просится одноцветный SVG-глиф.
- Не добавлять текст/цифры внутрь картинки — подпись делает верстка.
