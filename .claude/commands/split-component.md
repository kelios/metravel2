Распили файл `$1` на логические подкомпоненты. Поведение должно остаться идентичным.

План:
1. Прочитай файл целиком. Построй карту: какие секции JSX, какие handlers, какой state, какие effects.
2. Определи естественные границы (tabs, header/body/footer, modals, lists). Каждая ≥50 LOC и имеет самостоятельный смысл.
3. Предложи разбиение: список новых файлов с именами и зонами ответственности. **Жди подтверждения пользователя** перед извлечением.
4. После подтверждения:
   - Создай подкомпоненты в той же папке (или в `parts/` подпапке если их ≥3).
   - Пробрось props явно, избегай лишних контекстов.
   - Styles, выделенные только под подкомпонент, переноси вместе с ним.
   - Не трогай бизнес-логику, не рефакторь handlers — только извлечение.
5. Прогон `npm run typecheck && npm run check:fast`.
6. Если есть тесты на компонент — проверь что они проходят без изменений.

Не вводи новые абстракции (HOC, render props, generic wrappers). Простое разбиение → несколько маленьких файлов.
