description: Создать новую структурированную задачу в tasks/ по шаблону 000-template.md (через агента task-author)
---

Ты создаёшь новый TASK-файл в каталоге `tasks/` строго по формату проекта. Описание
задачи пользователь передаёт в `$ARGUMENTS` (свободный текст: баг, фича, идея, ссылка).

## Шаги

1. Прочитай `tasks/README.md` (правила формата) и `tasks/000-template.md` (шаблон).
2. Определи следующий номер `NNN`: просмотри `tasks/`, возьми `max(NNN) + 1` с ведущими
   нулями (`000-template.md` и `README.md` не считаются задачами).
3. Делегируй оформление агенту **task-author** (`Agent` tool,
   `subagent_type: "task-author"`). Передай ему:
   - сырое описание из `$ARGUMENTS`,
   - вычисленный `NNN` и сегодняшнюю дату (`YYYYMMDD` / `YYYY-MM-DD`),
   - требование: заполнить ВСЕ секции шаблона, проверяемые Acceptance Criteria + Gherkin,
     обязательный `## Task Contract` из `docs/TASK_BOARD_MCP.md`, корректный `## Assignment`
     (роли → реальные агенты проекта).
4. Если в `$ARGUMENTS` мало данных для проверяемых критериев приёмки — задай пользователю
   ОДИН компактный уточняющий вопрос, потом продолжай. Не выдумывай acceptance.

## Выход

Покажи: путь к созданному файлу `tasks/NNN-slug.md`, его ID/Status, кому назначена
(`Primary owner` / `Support`) и чего не хватает до готовности к старту (если что-то есть).

Аргументы: `$ARGUMENTS`
