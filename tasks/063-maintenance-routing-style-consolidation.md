## Goal

Normalize the routing in `maintenance/urls.py` to a single style (DRF Router + ViewSets with `@action` where needed) so all endpoints appear in the OpenAPI schema and the URL structure is predictable.

## Context

`maintenance/urls.py:19–31` mixes four incompatible styles:

- Lines 19–23: function-based views via bare `path()` — not visible to DRF's OpenAPI generator
- Lines 24–25: `ChatView`/`FeedbackView` as `APIView` with `as_view()` — partial OpenAPI coverage
- Line 30: `FilesView` via manual `as_view({'post':'file_upload_view'})` — non-standard ViewSet usage
- Line 31: Router for gallery — standard DRF Router

Additionally, `path('upload')` and `path('gallery/...')` are registered outside an `api/` namespace, inconsistently with other apps.

Function-based views are invisible to `drf-spectacular` (`SPECTACULAR_SETTINGS`), so image-serving and upload endpoints are missing from the OpenAPI schema consumed by frontends and tooling.
