# Бриф ревью квеста (контент → единая история)

Ты редактируешь ЖИВОЙ контент городского квеста `<QUEST_ID>` на проде metravel.by.
Цель — превратить квест из «гуляй и считай» в ЕДИНУЮ ИСТОРИЮ, где игрок —
участник, БЕЗ выдумок.

Сначала прочти `.claude/skills/metravel-quest/SKILL.md` (принципы, типы
answer_pattern, правило 4a: подсказка НЕ содержит ответа). Тон: русский, живой
человеческий, без AI-обвязки, без эмодзи, без канцелярита. Легенды — как легенды,
факты — как факты, всё ДОСТОВЕРНОЕ (сверяй веб-поиском).

## Шаг 0 — текущий контент
`curl -s "https://metravel.by/api/quests/by-quest-id/<QUEST_ID>/"` → распарси
intro + steps (поле `steps` — JSON-строка) + finale. У каждого шага: step_id,
title, location, story, task, hint, answer_pattern{type,value}, lat, lng,
maps_url, input_type, id.

## Шаг 1 — точки и координаты
Сверь, что lat/lng отвечают реальному названному объекту. Nominatim ИСПОЛЬЗУЙ
ЭКОНОМНО (≤1 запрос/сек, только для подозрительных точек):
`https://nominatim.openstreetmap.org/search?format=json&q=<место>, <город>`.
Если координата явно неверна (не тот объект/далеко >~150м) — включи исправленные
lat/lng. Задание должно быть выполнимо, стоя на точке.

## Шаг 2 — переписать как ОДНУ историю
- Найди сквозную нить (используй легенду/тему самого квеста — обычно она в
  названии). intro задаёт завязку и сквозной приём (проводник / тайна, которую
  игрок раскрывает; улики, ведущие к финалу). Игрок — «ты», действует, раскрывает
  историю шаг за шагом.
- `story` каждого шага: живой текст = реальная история + легенда (помечена как
  легенда) + топонимика, всё достоверно; ДВИГАЕТ сюжет (отсылка к предыдущему/
  следующему шагу), а не просто справка.
- `task`: НАБЛЮДАТЕЛЬНОЕ действие на месте, связанное с историей.
  **РАЗНООБРАЗЬ задания** — НЕ делай все «посчитай». В пределах квеста смешивай:
  посчитать (range); прочитать дату/имя/число на табличке/фасаде (exact или
  exact_any); назвать изображённый предмет/фигуру/материал (exact_any);
  найти конкретную деталь и назвать её (exact_any). Где объект позволяет —
  минимум 2–3 задания НЕ-счётные на квест.
- КРИТИЧНО (анти-лик): ОТВЕТ НЕ должен встречаться в `task` и `hint`. `hint` —
  только направление взгляда («подойди ближе», «подними взгляд», «смотри на
  фронтон»), НИКОГДА не сам ответ, его синоним или число/диапазон ответа.
- `answer_pattern` — корректный для наблюдаемого задания. Типы: `range`{min,max}
  — счёт; `exact` — одно известное число/слово; `exact_any` (JSON-массив строк,
  lowercase, с падежами/синонимами) — слово/название/предмет; `any_text`
  {min_length} — «опиши словами» (редко); `any` — «просто дойди/пауза».
  input_type: `number` для range/exact-числа/any_number, иначе `text`.

## Шаг 3 — самопроверка КАЖДОГО шага
Ответ (и ключевые токены answer_pattern.value) НЕ в task/hint. Если есть — перепиши.

## Шаг 4 — записать `scripts/review/<QUEST_ID>.json`
```
{ "intro": {"step_id":"...","story":"...","task":"...","hint":null,"answer_pattern":{...}},
  "steps": [{"step_id":"...","story":"...","task":"...","hint":"...","answer_pattern":{...},"lat":..,"lng":..}],
  "finale": {"text":"..."} }
```
Для каждого шага ВСЕГДА включай story+task+hint+answer_pattern. lat/lng — только
если исправлял. finale — можно отполировать, чтобы связать нить (необязательно).

## Шаг 5 — применить на прод
`node scripts/update-quest-content.js --quest-id=<QUEST_ID> --data=scripts/review/<QUEST_ID>.json --dry-run`
затем без `--dry-run`. (токен в `.secrets/metravel-token.json` уже есть.)

## Шаг 6 — верификация
`curl -s "https://metravel.by/api/quests/by-quest-id/<QUEST_ID>/"` → подтверди по
каждому шагу: токены answer_pattern.value НЕ в task/hint; story заметно богаче;
сквозная нить присутствует; разнообразие типов заданий.

## Отчёт (верни сжато)
Какая нить; по шагам (правки координат, убранные лики, тип answer_pattern,
счётное/не-счётное); итог верификации (сколько шагов, ликов — должно быть 0).
НЕ трогай другие квесты.
