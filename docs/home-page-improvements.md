# Главная страница (Home): BA‑анализ и план улучшений

## 1) Контекст и цель документа
Главная страница — это основной вход в продукт Metravel. Её задача — быстро объяснить ценность (“пишу путешествия → собираю книгу → делюсь”) и привести пользователя к ключевым действиям:

- создать первое путешествие (контент)
- найти вдохновение (поиск/каталог)
- собрать книгу (экспорт)

Документ фиксирует текущее состояние, проблемы UX/UI, гипотезы и приоритизированный backlog улучшений.

## 2) Целевые аудитории
- **Гость (не авторизован)**
  - хочет понять “что это” и попробовать без глубокого погружения
  - высокая чувствительность к доверию (приватность, примеры, отзывы)
- **Новый пользователь (зарегистрировался, 0 поездок)**
  - нужно дать быстрый старт: “написать первую историю”
  - нуждается в подсказках/шаблоне, чтобы не “застрять на пустом экране”
- **Активный пользователь (есть поездки)**
  - быстрый доступ к: “создать”, “поиск/вдохновение”, “экспорт/книга”
  - ожидает персонализацию: рекомендации, продолжить начатое

## 3) Основные сценарии (User Journeys)
### 3.1 Гость → понимание ценности → регистрация
1) видит заголовок + подзаголовок + primary CTA
2) смотрит “Как это работает”
3) видит примеры (вдохновение)
4) переходит к регистрации/логину

### 3.2 Новый пользователь (0 поездок) → создание первой истории
1) видит CTA “Написать первую историю”
2) переход на создание путешествия
3) успешное сохранение первой истории
4) подсказка: “добавь фото/заметки” или “собери книгу”

### 3.3 Активный пользователь → вдохновение / экспорт
1) видит CTA “Собрать книгу” или “Выбрать куда поехать”
2) быстро попадает в поиск/экспорт

## 4) Текущее состояние (по коду и UI)
### 4.1 Структура страницы
Сейчас Home собирается из:
- **Hero** (value proposition + 2 CTA)
- **Как это работает** (3 шага)
- **Вдохновение** (подборки: “в этом месяце”, “популярные направления”)
- **Final CTA**

### 4.2 Наблюдения по UI (скриншот)
- Верхняя навигация и Hero визуально “спокойные”, но важные элементы конкурируют за внимание.
- Блок “Как это работает” визуально читаем, но из-за больших вертикальных отступов пользователь “проваливается” вниз.
- Карточки выглядят аккуратно, но есть риск разной сетки/ширин на разных брейкпоинтах.

## 5) Проблемы и точки роста
### P0 (критично)
- **Неравномерная сетка/контейнеры**: разные блоки задают padding/maxWidth по-разному (часть через `ResponsiveContainer`, часть вручную).
  - эффект: визуальная “рваность”, неаккуратные края секций, сложнее поддерживать дизайн.

### P1 (важно)

### P2 (желательно)
- **Контентная ясность**:
  - часть формулировок можно сделать более “пользовательской” (в терминах пользы: “сохрани воспоминания”, “планируй” и т.п.).

## 6) Продуктовые гипотезы
- **H1**: Если сделать единый светлый, современный дизайн и одинаковую сетку, то повысится доверие и снизится bounce rate на Home.
- **H2**: Если добавить блок “Примеры книг/превью PDF + приватность”, то вырастет конверсия гостя в регистрацию.
- **H3**: Если показать “быстрый старт” для 0 поездок (шаги/шаблон/подсказки), то вырастет конверсия в создание первой поездки.

## 7) Улучшения (Backlog)
### P0 — Дизайн‑консистентность и читаемость
- **UI-правила (обязательно перед правками)**
  - см. `docs/ui-guidelines-ru.md` (иконки, плейсхолдеры, запрет эмодзи, обязательный прогон линтера/тестов)
- **Единый фон и “band”‑секции**
  - цель: ровный светлый фон, одинаковые отступы, аккуратные границы секций
- **Единый контейнер**
  - использовать `ResponsiveContainer` во всех секциях
  - убрать ручные `paddingHorizontal`/`maxWidth`, где возможно
- **Единые кнопки/интерактив**
  - состояния hover/focus, одинаковые радиусы/тени

## 8) Метрики успеха
### Основные
- **CTR primary CTA в Hero**
- **CTR secondary CTA (“Выбрать куда поехать”)**
- **Конверсия гость → регистрация/логин**
- **Конверсия новый пользователь → создание первой поездки**
- **Scroll depth** до блоков вдохновения/CTA

### Качественные
- NPS/опрос “насколько понятно, что делать дальше?”
- время до первого действия (TTFA)

## 9) Риски и ограничения
- **Данные для персонализации** могут отсутствовать (нужен fallback).
- **Производительность**: тяжелые секции (подборки/картинки) должны грузиться лениво.
- **Единая дизайн‑система**: важно не “разъехать” с остальными страницами.

## 10) Критерии готовности (Acceptance Criteria)
- Все секции на Home используют единый контейнер и согласованные отступы.
- Визуально единый светлый стиль (фон, радиусы, тени, типографика).
- На web и mobile нет горизонтального скролла, сетка карточек стабильна.
- Добавлены/описаны события аналитики для ключевых CTA.

## 11) Что уже сделано в коде (для трассируемости)
- Унифицирован фон Home и лёгкие отступы по вертикали.
- Приведены секции “Как это работает”, “Вдохновение” и финальный CTA к более светлой и согласованной подаче.
- Унифицирована ширина контейнера секций через `ResponsiveContainer maxWidth="xl"`.
- `HomeFinalCTA` переведён на `ResponsiveContainer` (убрана ручная логика `maxWidth/paddingHorizontal` внутри секции).
- Исправлена сетка “Вдохновение” на web:
  - неполная последняя строка не разъезжается из-за `space-between`
  - ширина карточек стабильна (3 колонки) и не растягивается при 1–2 карточках в строке
- Добавлен лёгкий блок доверия на Home (PDF/приватность/печать) сразу после Hero.
- Добавлен FAQ-блок на Home (аккордеон с короткими ответами).
