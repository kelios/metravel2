# 📱 Как выпустить Android-приложение MeTravel — пошагово для владельца

Это инструкция «делай раз, делай два». Не нужно быть программистом. Просто иди
по шагам сверху вниз. Где написано «выполни команду» — открываешь терминал и
вставляешь строчку. Где «зайди по ссылке» — открываешь ссылку в браузере.

> ⏱️ Сколько займёт: первый тестовый билд — ~30 минут (из них 15–20 ждёшь сборку).
> Полный путь до публикации — 1–2 дня (Google проверяет приложение).
> 💰 Сколько стоит: **разовый $25** за аккаунт разработчика Google Play. Больше ничего.

## 📖 Словарик (чтобы было понятно)

| Слово | Что это простыми словами |
|---|---|
| **EAS** | Облако компании Expo, которое СОБИРАЕТ приложение за тебя. Тебе не нужны Android Studio и прочее — нажал команду, оно собрало в интернете и дало ссылку на файл. |
| **APK** | Файл приложения, который можно скачать на телефон и установить вручную (как `.exe` на компьютере). Используем для теста. |
| **AAB** | Тот же файл приложения, но в формате, который требует Google Play для публикации. Используем для магазина. |
| **Google Play Console** | Личный кабинет в магазине Google, где ты управляешь приложением: описание, скриншоты, публикация. |
| **Service account (сервис-аккаунт)** | Специальный «робот-ключ» (файл), чтобы EAS сам загружал сборки в Google Play без твоего участия. Делается один раз. |
| **Internal testing (внутреннее тестирование)** | Закрытый канал в Google Play: приложение видно только тебе и тем, кого пригласишь. Сюда заливаем первым делом, проверяем — и только потом открываем всем. |
| **Терминал** | Чёрное окошко для команд. На Windows это **PowerShell**. Открой папку проекта `D:\metravel\metravel2` и работай там. |

## ШАГ 1. Завести Expo и войти в EAS

EAS — это то, что соберёт приложение. Нужно один раз войти.

1. Зайди по ссылке и зарегистрируйся (если аккаунта ещё нет):
   👉 **https://expo.dev/signup**
   (Запомни логин и пароль — это и есть `savran.juli` или новый.)

2. Открой терминал в папке проекта и выполни команду входа:
   ```powershell
   npx eas-cli login
   ```
   Введёшь почту/логин и пароль от Expo. Пароль в окне не отображается — это нормально, просто печатай и жми Enter.

3. Проверь, что вошёл:
   ```powershell
   npx eas-cli whoami
   ```
   Должно показать твой логин (например `savran.juli`). Если показывает `Not logged in` — повтори вход.

> ✅ После этого шага программист (я) уже смогу запускать сборки. Но ты можешь и сам — читай дальше.

## ШАГ 2. Сделать тестовый билд и проверить на телефоне (САМЫЙ ВАЖНЫЙ ШАГ)

Приложение раньше жило как сайт, на настоящем телефоне его ещё ни разу не запускали.
**Прежде чем платить Google и публиковать — надо убедиться, что оно вообще работает.**

1. Запусти сборку тестового APK (это бесплатно, собирается в облаке EAS):
   ```powershell
   npm run android:build:preview
   ```
   - EAS задаст пару вопросов при первом запуске (про ключ подписи — **соглашайся, выбирай вариант по умолчанию / "Generate new keystore"**, EAS хранит ключ за тебя).
   - Дальше жди ~15–20 минут. В конце терминал даст **ссылку на страницу сборки** и **ссылку на скачивание `.apk`**.

2. Установи APK на телефон:
   - Самый простой способ: на странице сборки (ссылка из терминала) есть **QR-код** — наведи камеру телефона, скачается APK, нажми «Установить».
   - Если телефон ругается «установка из неизвестных источников» — разреши для браузера/файлов (это нормально для теста).

3. **Проверь руками** (пройди по списку, отмечай галочки):
   - [ ] Приложение запускается, видно заставку и главный экран
   - [ ] Работают нижние вкладки (карта, места, профиль и т.д.)
   - [ ] **Карта** открывается, видны точки/маркеры, по тапу открывается место
   - [ ] Открывается статья-путешествие, листаются фото
   - [ ] Можно войти в аккаунт (логин)
   - [ ] Работает «Избранное» (сердечко)
   - [ ] Работает поиск
   - [ ] Спрашивает разрешение на уведомления / геолокацию когда нужно

4. **Если что-то падает или белый экран** — не страшно, так и должно ловиться на этом шаге.
   Запиши, ЧТО именно сломалось (на каком экране), и передай программисту — починим и пересоберём.
   **Пока этот шаг не пройден на зелёное — дальше (платить Google) не идём.**

> 💡 Для агентской проверки по кабелю используется локальная сборка:
> `cd android && ./gradlew :app:installDebug` или `:app:assembleDebug` +
> `adb install -r android/app/build/outputs/apk/debug/app-debug.apk`.
> `npm run android:build:dev` тратит EAS-сборку и запускается только по твоей явной команде.

## ШАГ 3. Завести аккаунт разработчика Google Play ($25)

Это личный кабинет, где будет жить приложение в магазине.

1. Зайди:
   👉 **https://play.google.com/console/signup**
2. Войди своим Google-аккаунтом (почта Gmail).
3. Выбери тип аккаунта:
   - **Personal (личный)** — проще, подходит, если публикуешь как частное лицо.
   - **Organization (организация)** — если есть юрлицо/ИП (потребуют подтверждение).
   Для старта обычно берут **Personal**.
4. Заполни данные (имя, адрес, телефон) и **оплати разовые $25** картой.
5. Google может попросить **подтвердить личность** (фото документа) — это нормально, сделай по их инструкции. Иногда проверка занимает 1–2 дня.

> ⚠️ С 2023 года Google для НОВЫХ личных аккаунтов иногда требует «тестирование с 12 тестировщиками 14 дней» перед публикацией. Если у тебя это требование появится — мы как раз используем **внутреннее тестирование** (Шаг 8), оно для этого и нужно. Просто закладывай +2 недели на этот период.

## ШАГ 4. Создать приложение в Google Play Console

1. Открой консоль:
   👉 **https://play.google.com/console**
2. Нажми **«Create app» / «Создать приложение»**.
3. Заполни:
   - **App name (название):** `MeTravel — путешествия`
   - **Default language:** Русский (ru-RU)
   - **App or game:** App (приложение)
   - **Free or paid:** Free (бесплатное)
   - Поставь галочки-подтверждения (правила, экспортные законы).
4. Нажми **Create**.

> На этом шаге Google ещё НЕ знает имя пакета `by.metravel.app` — он привяжется
> автоматически, когда мы зальём первую сборку (Шаг 8). Ничего вручного не нужно.

## ШАГ 5. Сделать «робот-ключ» (service account), чтобы заливать сборки автоматически

Это нужно, чтобы команда `submit` сама отправляла приложение в Google Play.
Делается **один раз**. Звучит страшно, но это просто несколько кнопок.

📘 Официальная пошаговая инструкция с картинками (открой её рядом и иди параллельно):
👉 **https://docs.expo.dev/submit/android/#creating-a-google-service-account-key**

Кратко, что там:
1. В **Google Play Console** → внизу слева **Setup → API access** (Настройки → Доступ к API).
2. Нажми создать новый сервис-аккаунт → тебя перекинет в **Google Cloud Console**.
3. В Google Cloud создаёшь Service Account, затем у него **создаёшь ключ → формат JSON** → скачается файл `*.json`.
4. Вернись в Play Console → дай этому сервис-аккаунту права (роль **Release / Admin** достаточно) на твоё приложение.

5. **Куда положить скачанный файл-ключ** (важно!):
   - Переименуй файл в **`google-play-service-account.json`**
   - Положи его **в корень проекта**: `D:\metravel\metravel2\google-play-service-account.json`
   - Этот путь уже прописан в настройках и **уже защищён** — файл НЕ попадёт в git (проверено).
   - ❗ Никому не пересылай этот файл и **не вставляй его содержимое в чат** — это секретный ключ.

> Проверить, что ключ лежит правильно и не утечёт, можно командой:
> ```powershell
> git check-ignore google-play-service-account.json
> ```
> Если в ответ напечаталось имя файла — всё хорошо, он защищён.

## ШАГ 6. Подготовить описание и картинки для магазина

Текст уже написан — лежит в файле **`docs/ANDROID_STORE_LISTING.md`** (в этом же проекте).
Оттуда копируй готовое:

- **Название, краткое и полное описание** (на русском) — копируй как есть, при желании поправь тон.
- **Privacy Policy URL** — ссылка на политику конфиденциальности. Укажи `https://metravel.by/privacy` (проверь, что страница открывается).
- **Data Safety (Безопасность данных)** — в файле есть готовая табличка, что заполнять (геолокация, фото, аналитика). Это обязательная анкета в Play Console.

**Картинки, которые надо приложить** (в Play Console → Main store listing):
- [ ] **Иконка 512×512** — есть в проекте (`assets/images/icon.png`), проверь размер.
- [ ] **Feature graphic 1024×500** — большая «шапка». Её нужно нарисовать (Canva/дизайнер).
- [ ] **Скриншоты телефона** — минимум 2, лучше 4–6. Просто сделай скриншоты приложения на телефоне (карта, путешествие, квест, список) из Шага 2.

📘 Где в консоли это заполнять: левое меню → **Grow → Store presence → Main store listing**.

## ШАГ 7. Собрать «магазинную» версию (AAB)

Когда тест (Шаг 2) прошёл и аккаунт Google готов — собираем финальную версию.

1. (Рекомендуется) сначала общая проверка проекта:
   ```powershell
   npm run release:check
   ```
2. Собрать AAB для магазина:
   ```powershell
   npm run android:build:prod
   ```
   Снова ~15–20 минут в облаке. На выходе — файл **`.aab`** и ссылка на него.
   Номер версии (versionCode) поднимется автоматически — вручную ничего менять не надо.

## ШАГ 8. Залить в Google Play (внутреннее тестирование)

1. Отправить последнюю собранную версию в магазин:
   ```powershell
   npm run android:submit:latest
   ```
   Эта команда возьмёт AAB из Шага 7 и зальёт его в канал **internal (внутреннее тестирование)** —
   это уже прописано в настройках, ничего выбирать не надо.

2. Зайди в **Play Console → Test → Internal testing**:
   👉 **https://play.google.com/console** → твоё приложение → раздел Testing → Internal testing.
   - Добавь себя (и пару друзей) в список тестировщиков по почте.
   - Скопируй **ссылку для тестировщиков**, открой её на телефоне, прими приглашение, установи приложение из Google Play.

3. **Проверь приложение из магазина** (не APK, а настоящую установку из Play) — те же пункты, что в Шаге 2.

> На этом этапе Google может ещё показывать «приложение на проверке» — подожди, пока станет доступно (от часа до суток).

---

## ШАГ 9. Открыть приложение всем (Production)

Когда внутреннее тестирование прошло хорошо:

1. В Play Console заполни всё, что Google требует перед публикацией (он покажет красные галочки в разделе **Dashboard / Publishing overview**):
   - [ ] Store listing (Шаг 6) — заполнено
   - [ ] **Content rating** (анкета возрастного рейтинга) — пройди, для путешествий обычно «для всех»
   - [ ] **Data safety** (Шаг 6) — заполнено
   - [ ] **Target audience** (целевая аудитория) — взрослые
   - [ ] **App access** — если для части функций нужен логин, дай Google тестовый аккаунт (можно e2e-аккаунт)
   - [ ] Privacy Policy URL — указан

2. Когда все галочки зелёные — в разделе **Production** нажми **Create new release**,
   добавь туда ту же сборку (можно «promote» из internal в production — кнопка прямо в консоли),
   и нажми **Review release → Start rollout to Production**.

3. Google проверяет приложение (обычно от нескольких часов до 1–3 дней). После одобрения оно появится в Google Play по поиску «MeTravel». 🎉

## 🆘 Если что-то пошло не так (частые проблемы)

| Проблема | Что делать |
|---|---|
| `npx eas-cli whoami` пишет **Not logged in** | Повтори `npx eas-cli login`. Проверь почту/пароль на expo.dev. |
| Сборка падает с ошибкой про **keystore** | При вопросе выбирай «Generate new keystore» — пусть EAS создаст и хранит ключ сам. |
| Приложение **не ставится** на телефон (APK) | Разреши «установку из неизвестных источников» для браузера. Или ставь через Google Play (Шаг 8). |
| **Белый экран / краш** при запуске | Запиши экран, где это случилось, и передай программисту — починим и пересоберём (`build:preview`). |
| `submit` пишет, что **нет доступа** к приложению в Play | Проверь, что сервис-аккаунту (Шаг 5) выданы права в Play Console на это приложение. |
| Google требует **«12 тестировщиков 14 дней»** | Это новое правило для личных аккаунтов. Используй Internal testing (Шаг 8) и подожди срок — потом откроется Production. |
| Не знаю, куда положить **ключ** | В корень проекта, имя ровно `google-play-service-account.json`. Проверка: `git check-ignore google-play-service-account.json`. |
