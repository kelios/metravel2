# Спецификация: Календарь путешествий («Мой календарь»)

Экран `app/(tabs)/calendar.tsx`. Источник данных — стор `stores/travelStatusStore.ts`
(`entries`). Раскладка и статусы — `utils/travelCalendarDate.ts`,
`utils/travelStatusCalendarDisplay.ts`, `components/screens/calendar/*`.

## Термины
- **Явная дата** — сохранённое поле статуса (`visitedDate` / `plannedDate` /
  `wishlistDate`) в валидном формате `YYYY-MM-DD`.
- **Вычисленная дата** — дата, рассчитанная из года+месяца, когда явной нет.
- **Мои путешествия** — все travel, где автор = текущий пользователь:
  опубликованные, на модерации И черновики (загружаются с `includeDrafts:true`
  + токен).
- **Свободный выходной** — суббота/воскресенье месяца, ещё не занятая другой
  поездкой.
- **moderationState** — `draft` (черновик, `publish=0`) / `pending`
  (на модерации, `publish=1 & moderation=0`); опубликованные — `undefined`
  (бейджа нет).

---

## Группа A. Источник данных: показываются ВСЕ мои путешествия

**A1 — Опубликованное путешествие попадает в календарь.**
Дано: 1 опубликованное путешествие (`publish=1, moderation=1`). Когда: открыт
экран. Тогда: присутствует в списке и отмечено точкой на сетке.

**A2 — Черновик попадает в календарь.**
Дано: путешествие-черновик (`publish=0`), автор = текущий пользователь. Тогда:
отображается наравне с опубликованными, помечено бейджем «Черновик».

**A3 — Путешествие на модерации попадает в календарь.**
Дано: `publish=1, moderation=0`. Тогда: отображается, помечено бейджем
«На модерации».

**A4 — Чужие путешествия НЕ попадают.**
Дано: опубликованное путешествие другого автора без явного статуса
пользователя. Тогда: в календаре его нет.

**A5 — Пустой список.**
Дано: ни одного путешествия и явного статуса. Тогда: EmptyState, сетка без
точек, ошибок нет.

**A6 — Неавторизованный пользователь.**
Дано: пользователь не залогинен. Тогда: EmptyState «Войдите», запросов за
путешествиями нет.

**A7 — Дедупликация.**
Дано: одно путешествие пришло и как авторское, и как явный статус. Тогда: ровно
одна запись (явный статус в приоритете, метаданные добираются из авторского).

---

## Группа B. Раскладка по указанной дате

**B1 — Есть явная дата → ставим ровно на неё.**
`visitedDate = 2026-03-14` → точка на 14.03.2026; клик по 14.03 показывает
поездку.

**B2 — Явная дата — якорь, её не двигают.**
Две поездки с одинаковой явной датой стоят на этом дне (явные даты не подлежат
анти-коллизии).

**B3 — Явная дата вне текущего месяца сетки.**
Явная `2026-07-20`, открыт март → на мартовской сетке точки нет; в списке запись
видна всегда; на июльской — точка на 20.07.

**B4 — focusDate.**
Самая ранняя поездка в мае 2026 → сетка автоматически открывается на мае 2026.

---

## Группа C. Fallback: нет даты, но есть месяц+год → выходные

**C1 — Год + месяц → выходной.**
Без явной даты, `travelYear=2026, travelMonth=август` → вычисленная дата —
суббота/воскресенье августа 2026 (выходные приоритетны).

**C2 — Детерминизм.**
Повторный рендер/перезаход → та же дата (seed = id, без рандома между сессиями).

**C3 — Все выходные заняты → будни.**
Если все выходные месяца заняты, новая бездатовая поездка встаёт на свободный
будний день того же месяца.

**C4 — Год есть, месяц не распознан.**
`travelYear=2026`, месяц пустой/мусорный, `allowYearOnly=true` → месяц выбирается
детерминированно по seed, дата — выходной этого месяца.

**C5 — Русские и английские названия месяцев.**
`«март»` / `«March»` / `«03»` / `{id:3}` → все распознаются как март.

**C6 — Клик по вычисленной дате матчит по месяцу.**
Для вычисленных дат клик по любому дню месяца показывает поездку (совпадение по
году+месяцу, а не по дню).

---

## Группа D. Свободные выходные / анти-коллизия («смотреть что было за месяц»)

**D1 — Две бездатовые поездки в одном месяце → разные выходные.**
Обе `2026-06`, без явной даты → встают на разные свободные выходные июня.

**D2 — Явная дата занимает слот для вычисленных.**
A — явная `2026-06-06` (сб); B — без даты в июне → B берёт следующий свободный
выходной, не 6 июня.

**D3 — Порядок раздачи стабилен.**
Распределение детерминировано и стабильно между рендерами (порядок по id).

**D4 — Переполнение месяца.**
Бездатовых поездок больше, чем дней → раздаются все дни (выходные → будни), без
краша; поездка не пропадает.

---

## Группа E. Пустые / некорректные даты

**E1 — Нет ни явной даты, ни года.**
На сетку не ставится (точки нет), но в списке присутствует и сортируется в
конец. Не теряется.

**E2 — Невалидный год (вне 1900–2200).**
`travelYear=3000` → как отсутствие года (см. E1).

**E3 — Битая явная дата.**
`«2026-13-40»` / `«вчера»` / пусто → невалидная строка игнорируется, включается
fallback по году+месяцу (Группа C).

---

## Группа F. Статус выводится из даты (load-bearing)

Статус авторского путешествия вычисляется из эффективной даты (явной ИЛИ
вычисленной), если пользователь не задал статус вручную.

**F1 — Явная дата в прошлом → «Был».**
Дата `2025-09-10`, сегодня `2026-07-10` → `visited`.

**F2 — Явная дата в будущем → «Планирую».**
Дата `2026-12-20` → `planned`.

**F3 — Дата = сегодня → «Планирую».**
Граница «сегодня и позже» = будущее → `planned`.

**F4 — Нет даты вообще → «Планирую».**
Ни даты, ни года («хочу куда-то») → `planned`.

**F5 — Вычисленная дата в прошлом → «Был».**
Явной нет, `travelYear=2024, travelMonth=май` → вычисленный выходной мая 2024
(прошлое) → `visited` (статус считается по итоговой дате, не только по явной).

**F6 — Вычисленная дата в будущем → «Планирую».**
`travelYear=2027, travelMonth=март`, явной нет → `planned`.

**F7 — Пользователь вручную переопределил статус.**
Прошлая дата, но пользователь отметил «Хочу» (`wishlist`) → ручной статус в
приоритете над авто-выводом, остаётся `wishlist`.

**F8 — Смена даты меняет статус.**
Черновик «Планирую» (будущая дата); пользователь меняет дату на прошлую →
авто-статус пересчитывается в «Был» (если статус не зафиксирован вручную).

### Таблица решений статуса

| Есть явная дата | Дата | Ручной статус | Итог |
|---|---|---|---|
| да | < сегодня | нет | Был |
| да | ≥ сегодня | нет | Планирую |
| нет, есть год+месяц | вычислено < сегодня | нет | Был |
| нет, есть год+месяц | вычислено ≥ сегодня | нет | Планирую |
| нет вообще | — | нет | Планирую |
| любое | любое | есть (Был/Планирую/Хочу) | ручной статус |

---

## Группа G. Бейдж модерации

**G1 — Черновик → бейдж «Черновик».**
`moderationState='draft'` → пилюля warning-палитры (иконка `edit-3`) в углу над
фото, кадр не затемняется.

**G2 — На модерации → бейдж «На модерации».**
`moderationState='pending'` → пилюля info-палитры (иконка `clock`).

**G3 — Опубликованное → без бейджа.**
`moderationState=undefined` → бейджа нет.

**G4 — Отсутствие полей publish/moderation в ответе API.**
Поля не пришли → трактуется как опубликованное (бейджа нет), чтобы не пометить
черновиком всё подряд.

---

## Группа H. Сетка MiniCalendar

**H1 — Навигация по месяцам** сохраняет корректные точки для каждого месяца.

**H2 — Точка в дне** появляется, если хотя бы одна поездка (явная или
вычисленная) попадает на этот день/месяц по правилам групп B–C.

**H3 — Выбор дня** фильтрует список (B1/C6); повторный клик снимает фильтр.
