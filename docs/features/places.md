# Фича: places

**Последняя актуализация:** 2026-07-15
**Ответственный домен:** frontend map/places

## TL;DR

`/places` - каталог отдельных точек сайта, а не путешествий. Пользователь выбирает страну,
категорию или поиск, видит карточки мест, открывает конкретную точку на карте или переходит
к путешествию, где эта точка используется.

## Точки входа

| Путь | Назначение |
| --- | --- |
| `app/(tabs)/places.tsx` | route/tab entry |
| `screens/tabs/PlacesScreen.tsx` | экран каталога, фильтры, пагинация, переходы |
| `api/places.ts` | загрузка широкого набора точек через map API |
| `utils/placesCatalog.ts` | нормализация точек, стран, категорий, заголовков |
| `components/places/PlaceListCard.tsx` | карточка места |

## Контракт продукта

- Показываем именно places/точки из координат путешествий, не карточки путешествий.
- По умолчанию показываем полезную подборку: замки, дворцы, экотропы, колесо обозрения,
  водохранилища, озера, реки и ручьи.
- Фильтр стран находится выше категорий и влияет на категории, счетчики и список мест.
- `Все` показывает полный каталог, но UI не должен рендерить все карточки сразу:
  список раскрывается порциями по `20` мест при скролле/кнопке дозагрузки.
- Карточка места должна иметь два основных действия:
  - `На карте` - открыть `/map` с координатами точки и малым радиусом фокуса;
  - `Прочитать` - открыть внутренний travel route, а для внешнего URL использовать
    только helpers из `utils/externalLinks.ts`.

## Данные

`fetchPlacesCatalog` переиспользует `fetchTravelsForMap` с большим радиусом и нормализует
ответ через `normalizeCatalogPlaces`.

Нормализация обязана:

- отбрасывать записи без валидных координат;
- дедуплицировать точки;
- строить читаемый `title` из имени или адреса;
- не использовать технические подписи вроде `12`, `№7`, `136км` как название карточки;
- определять страну из явных полей или адреса.

## UI-правила

- Использовать существующие `Button`, `Chip`, `ImageCardMedia` и дизайн-токены.
- Не добавлять emoji-иконки и декоративные плейсхолдеры.
- Состояния loading/error/empty/list должны иметь стабильную геометрию без скачков.
- Карточка места должна использовать тот же point/place template, что popup
  карты и карточка точки в travel details: доминирующее фото, затем название,
  координаты с copy, переход к связанному путешествию/странице, раскрываемая
  навигация и "мои точки" при доступности.
- Если у карточки/попапа есть координаты, набор навигации не сокращается до
  Google: показывай Google Maps, Apple Maps, Organic Maps/offline, Waze,
  Яндекс Карты, Яндекс Навигатор и OpenStreetMap. Telegram/share только
  дополнительное действие.
- Для мест, связанных с travel, статус должен быть явным: "Был здесь",
  "Хочу поехать", "Планирую" или компактная кнопка "Был / Хочу / Планирую".
- Mobile web, Android и iOS не должны расходиться визуально или по поведению
  карточки места; платформенные отличия допустимы только для safe area, map
  engine и нативных API.
- На web проверять страницу в браузере после видимых изменений.

## Проверки

Для изменений только в каталоге мест:

```bash
npm run check:fast
```

До него запускаются ближайшие tests для `api/places.ts`,
`utils/placesCatalog.ts` и `PlaceListCard` по затронутому scope. Видимый web UI
проверяется в браузере; native point/place flow — на локально установленной
Android-сборке.

Если меняются переходы `На карте` / `Прочитать`, дополнительно:

```bash
npm run guard:external-links
npm run governance:verify
```
