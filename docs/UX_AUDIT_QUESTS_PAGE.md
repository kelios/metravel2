## 1. Описание страницы

### Цель
Каталог городских квестов — интерактивных прогулок по точкам с загадками и заданиями.

### Целевая аудитория
- Туристы — ищут активности в новом городе
- Местные жители — хотят исследовать свой город
- Семьи с детьми — ищут развлечения на выходные
- Молодёжь 18-35 — любители геймификации

### Основной сценарий
1. Пользователь заходит на страницу квестов
2. Выбирает город/локацию
3. Просматривает доступные квесты
4. Переходит к прохождению

## 3. Информационная архитектура

### Текущая структура
```
Header → Hero → Фильтры (sidebar/drawer) → Карточки квестов → Footer
```

### Отсутствующие блоки
- Объяснение формата (что такое квест)
- Social proof (отзывы, количество прохождений)
- Превью маршрута на карточке
- Фильтры по сложности/времени
- Рекомендации («Популярные», «Новые»)

### Лишние блоки
- Дублирующий select на desktop
- Статистика «Сейчас в подборке: 1»

## 4. UX-проблемы (детально)

### Критические (High)

**1. Требуется авторизация для просмотра квеста**
- При клике на карточку → экран «Войдите, чтобы начать квест»
- Пользователь не может оценить квест до регистрации
- **Влияние:** -40-60% конверсии

**2. Карта квестов пустая**
- /quests/map показывает карту без маркеров
- Функция «Карта» бесполезна
- **Влияние:** Потеря пользователей географического поиска

**3. Нет value proposition**
- Непонятно зачем проходить квесты
- Высокий bounce rate
- **Влияние:** -30-50% конверсии

### Средние (Medium)

**4. Автовыбор города Kraków**
- Пользователь из Минска видит квесты Кракова по умолчанию

**5. «Рядом» требует геолокацию без объяснения**
- Показывает «гео отключена» без инструкции

**6. Нет фильтров по сложности/времени**

**7. Карточка квеста минималистична**
- Нет описания, превью маршрута, отзывов

## 5. UI-дизайн

### Положительное
- Типографика читаемая (28-36px заголовки)
- Достаточный контраст текста
- Адекватные отступы

### Проблемы
- Обложка квеста тёмная — сложно разглядеть
- Badge «Лёгкий» мелкий (11px)
- Select на desktop не стилизован
- Нет явного hover-эффекта на карточке

---

## 6. Карта квестов

### Критическая проблема
**Карта (/quests/map) не показывает маркеры квестов.**

### Что должно быть
- Маркеры всех квестов
- Popup с превью при клике
- Фильтры по сложности/времени
- Легенда маркеров

---

## 7. Мобильный UX

### Положительное
- Адаптивная вёрстка работает
- Фильтры в drawer
- Bottom tabs навигация

### Проблемы
- Фильтры скрыты (требуют клика)
- Карточка занимает много места (1 квест на экране)
- «Квесты» нет в bottom tabs (через «Ещё»)
- Некоторые touch targets < 44px

---

## 8. Конверсионная логика

### Текущая воронка
```
Посещение → Выбор города → Клик на карточку → БЛОКЕР: авторизация → Квест
```

### Рекомендуемая воронка
```
Посещение → Выбор города → Клик → Превью (без авторизации) → «Начать» → Авторизация → Квест
```

### Отсутствует
- Описание выгод
- Social proof
- Micro-conversions (избранное, поделиться)

## 10. План редизайна

### Новая структура
```
Hero (value proposition + CTA)
  ↓
Quick Filters (город + сложность + время) — sticky
  ↓
Quest Cards Grid (с рейтингами и превью маршрута)
  ↓
«Как это работает» (onboarding)
  ↓
Social Proof (отзывы, статистика)
  ↓
Footer
```

### Улучшения карты
1. Показывать маркеры квестов
2. Кластеризация близких квестов
3. Popup с превью при клике
4. Фильтры на карте
5. Легенда по сложности
