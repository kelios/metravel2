## 9. ИНДЕКСАЦИЯ САЙТА — корень проблемы и план фикса (2026-06-08)

### Диагноз
GSC «Индексирование страниц»: **422 в индексе / 444 НЕ в индексе** (≈половина сайта), тренд
непроиндексированных РАСТЁТ. Доминирующая причина — **269 «просканирована, но пока не
проиндексирована»** (61 % всех проблем). Прочее (94 canonical-варианта, 31 редирект и т.п.) —
в основном норма.

**Корневая причина найдена (проверено curl'ом прод-HTML):**
статические SEO-страницы отдают только `<title>` + `<meta>` + **скелетон-заглушку**
(`scripts/ssg-skeletons.js`: серые полоски `.ssg-travel-line`, hero-картинка, скрытый `<h1>`).
**Текста статьи в HTML НЕТ** — он подгружается JS как JSON. Googlebot по сырому HTML видит
≈20 видимых слов → считает страницу малоценной → не индексирует. Часть страниц (напр.
`/travels/dominikana`) отдаёт даже generic-заголовок «Путешествие | Metravel» — SSG для них
не отработал.

**Вторичные факторы:** раздутые keyword-stuffed заголовки/слаги (158 шт. `title-too-long`),
слабая внутренняя перелинковка, медленный LCP (FE-1). (Баг слага «zagolovok-» НЕ подтверждён.)

**Факт по статьям АВТОРА (user_id 1, URL Inspection API, 2026-06-08):** всего 294, в индексе
**238 (81 %)**, не в индексе 56 (19 %): из них **37 «URL неизвестен Google»** (Google ещё не
открыл — почти все свежие, нужен sitemap + перелинковка + «Запросить индексирование») и
**19 «просканирована, но не проиндексирована»** (нужен FE-IDX-1 + углубление контента).
То есть сайтовые 444 непроиндексированных — это в основном ДРУГОЙ автор + canonical-варианты,
а у Julia ситуация заметно лучше. Baseline: `scripts/.index-cache/baseline-2026-06-08.json`.

> Вывод: главный рычаг индексации — **не переписывание текста, а серверный рендер тела
> статьи в статический HTML** (FE-IDX-1). Контент-правки помогают сниппету/CTR, но в индекс
> не пустят, пока Google не видит тело.

### Контент (агент `index-doctor`, scope = ТОЛЬКО user_id 1)
Мониторинг: `npm run stats:index` / `stats:index:json` (`scripts/index-status.js`, GSC URL
Inspection API). Агент чинит тонкий контент, лиды, внутренние ссылки и выдаёт BE-worklist
(мусорные слаги). **Честно:** до FE-IDX-1 контент-правки слабо двигают индекс — это
подготовка почвы + рост сниппета/CTR у уже проиндексированных.

### Порядок исполнения
1. **FE-IDX-1 + FE-IDX-2** (код) — главный рычаг, разблокирует индексацию всего сайта.
2. Деплой → OWN-IDX-1 (запросить индексирование) → ждать переобхода.
3. Параллельно: `index-doctor` по статьям user_id 1 (контент/ссылки) + BE-IDX-2 (мусорные слаги).
4. BE-IDX-1 (редиректы) → сокращение заголовков → FE-IDX-3 (перелинковка).
