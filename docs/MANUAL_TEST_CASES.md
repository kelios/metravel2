## Прогон по платформам

После написания/ревью всех сценариев выше — выполнить прогон в двух средах. Один и тот же ID проверяется на обеих платформах; различия из колонки «Платформенные отличия» учитывать.

### Чек-лист платформ

| Платформа | Среда | Особенности проверки |
|-----------|-------|----------------------|
| **Web Desktop** | Браузер ≥1440px | 2–3 колонки списка, sidebar фильтров, Leaflet-карта, hover/focus, Web Share API, экспорт PDF/GPX/KML |
| **Web Mobile** | Браузер ≤600px | 1 колонка, фильтры в fullscreen modal, bottom sheet на карте, fullscreen popup места |
| **iOS** | Симулятор/устройство | React Native Maps, нативный share/picker, haptic feedback, KeyboardAvoidingView |
| **Android** | Эмулятор/устройство | React Native Maps, нативный share/picker, кнопка «назад» системы |

### Матрица результатов

> Прогон от 2026-06-01, окружение: Expo Web dev (`localhost:8081`), API-прокси → локальный бэкенд `http://192.168.50.36`, аккаунт: e2e (авторизация не дошла — см. блокер). iOS/Android из этой среды недоступны → `verify pending`.

**Блокер окружения:** Expo Web dev-сервер (Metro) воспроизводимо падает (OOM) при on-demand бандлинге тяжёлого роута `/search` и при навигации между роутами — машина под нехваткой памяти (swap ~5.8G/7G). Главная (`/`) и единичный рендер `/search` отрабатывают, но интерактивные шаги, требующие переходов между экранами, прерываются крашем + рестарт ~90с. Полный интерактивный прогон в dev-режиме нестабилен.
**Рекомендация:** для стабильного QA собрать прод-бандл (`npm run build:web`) и отдавать статикой через `E2E Web Server` (8085) — без on-demand бандлинга Metro краши уходят.

### Кросс-функциональные проверки (на обеих платформах)

- [ ] Гость vs авторизованный: действия (избранное, статус, комментарии, создание) корректно ограничены/доступны
- [ ] Глубокие ссылки: прямой переход по URL деталей/карты работает
- [ ] Производительность: нет фризов при скролле списка и панорамировании карты
- [ ] Консоль/логи: нет ошибок и необработанных промисов
- [ ] Локализация: тексты на нужном языке, без «дыр» в переводах
- [ ] Адаптив: проверить breakpoints 600 / 1024 / 1440 px
