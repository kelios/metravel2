## 2. Демо-сценарий (10–15 минут, production web)

Все URL проверены GET-пробой 2026-07-05 (HTTP-статусы и содержимое — §4). Показывать с **мобильного вьюпорта или телефона** — 78% аудитории мобильные (ANALYTICS_AUDIT §1).

### Шаг 0 (подготовка, до звонка)
- Открыть все вкладки заранее; прогреть страницы (первый заход после деплоя может ждать прогрева tile-кэша карты).
- Брать slug'и статей ТОЛЬКО из живой выдачи каталога: старые переименованные slug'и отдают заглушку «Страница переехала» (проверено: `/travels/krakov-karer-zakshuvek` → stub, это корректный SEO-механизм 301-переезда, но в демо выглядит как битая страница).
- Прогнать этот сценарий целиком в день демо (чеклист §6).

### Мин 0–2 — Главная: что это за продукт
`https://metravel.by/` (200, 0.47 c)
- Hero + подборки маршрутов, поиск. Тезис: «контент-платформа выходного дня для RU/BY: готовые маршруты с картой и фото».
- Сказать про 3 платформы: web + Android + iOS из одной кодовой базы (Expo/React Native).

### Мин 2–6 — Статья-путеводитель: ядро SEO-трафика
`https://metravel.by/travels/minsk-za-vykhodnye-putevoditel-po-stolitse-belarusi` (200, 0.92 c)
- Показать: структура путеводителя, авторские фото, точки на карте внутри статьи, блок «Похожие путешествия», блок квеста города.
- **Evidence-тезис:** это не SPA-пустышка — тело статьи в сыром HTML (проверено: 32 `<p>`, meta description, JSON-LD, crawlable перелинковка). Именно поэтому Google даёт +42% кликов за месяц.
- Назвать масштаб: 389 опубликованных статей, 464 страницы уже получают клики из Google.

### Мин 6–9 — Карта: интерактивная ценность
`https://metravel.by/map` (200, 0.51 c)
- Поиск мест рядом, фильтры, карточка точки с фото, ссылка в навигатор.
- Тезис: карта превращает читателя в планировщика — «нашёл статью → собрал свою поездку».

### Мин 9–12 — Квесты: игровой слой и retention-механика
`https://metravel.by/quests` (200, 0.58 c)
- Хаб: 53 квеста в 39 городах (Минск, Краков, Варшава, Вильнюс, евростолицы…). Открыть один квест, показать точки-задания.
- Честный тезис: квесты — готовый продукт с нулевым пока маркетингом (0 кликов из Google за 3 мес — ANALYTICS_AUDIT §2.1); задача спринта — SEO-видимость (#770) и гостевой вход. Это upside, а не traction.

### Мин 12–14 — Регистрация и личная ценность аккаунта
`https://metravel.by/registration` (200, 0.51 c)
- Показать форму и объяснить ценность аккаунта: избранное, свои маршруты, прогресс квестов, PDF-книга путешествий.
- **Не обещать конверсию** — регистраций сейчас 0/мес (ANALYTICS_AUDIT §1); показываем механику + план активации (#768 воронка-цели, #769 CTA + deferred-регистрация).
- Бонус-слайд вживую: `https://metravel.by/roulette` (200) — «рулетка маршрутов» как пример продуктовых фич.

### Мин 14–15 — Экономика и ask
- Сегодня: выручка 0, монетизация сознательно отложена до роста трафика (правило GROWTH_PLAN: «деньги — следствие роста»).
- Следующие шаги монетизации УЖЕ заведены и спроектированы: sponsored-места на карте/в выдаче — board **#620** (BE-поля `is_sponsored`) + **#35** (FE-маркер и sort-boost); аффилиат-каналы Travelpayouts (FE-код готов, ждёт ID владельца — GROWTH_PLAN FE-2/OWN-9); премиум-PDF и премиум-квесты (GROWTH_PLAN §7).
- Valuation-рамка — §5.

## 5. Valuation assumptions (с caveats)

**Жёсткое правило этого раздела:** подтверждённой выручки нет (0 — GROWTH_PLAN §0), поэтому никакая revenue-based оценка сейчас не является фактом. Ниже — три честные линзы и входные параметры; итоговый range требует данных владельца и рынка — помечено TBD.

### Линза A — актив «контент + SEO-трафик» (замещение/сравнимые)
- Актив: 389 SSG-статей (полное тело в HTML, JSON-LD, перелинковка) + 53 квеста в 39 городах + фотоархив автора.
- Traction-мультипликатор: organic-клики +42% за месяц без бюджета; вовлечённость (12 мин / 7.1 стр / 10% отказов) значительно выше типичной для контент-сайтов.
- Метод: цена сравнимых контент-сайтов = множитель к месячной выручке или к трафику. Входные данные (сравнимые сделки RU/BY-ниши, cost-per-article замещения): **TBD (owner)**.

### Линза B — replacement cost (кодовая база)
- Продукт: кросс-платформенное приложение (React 19 / RN / Expo, web SSG + Android + iOS), карта (Leaflet/RN Maps), квест-движок, PDF-генератор книг, admin/moderation-контур, e2e/Jest-инфраструктура.
- Метод: человеко-месяцы воссоздания × ставка. Оценка человеко-месяцев и ставки: **TBD (owner)** — не фиксируем без согласованной методики.

### Линза C — forward revenue (только как план, не факт)
- Заведённые каналы: sponsored-места (#620 BE + #35 FE; pipeline продаж — GROWTH_PLAN OWN-6/OWN-7: 5 партнёров × 20–50 BYN как первая цель), Travelpayouts-аффилиат (FE-код готов, ждёт ID — FE-2/OWN-9), премиум-PDF (FE-8), премиум-квесты (§7 GROWTH_PLAN).
- Всё это — **plan, не traction**: 0 заказов аффилиата за 9 мес (Belkraj), 0 sponsored-контрактов.

### Caveats (проговаривать до того, как спросят)
1. Выручка = 0; регистрации = 0/мес; активация не построена (в работе — #768/#769).
2. Абсолютный трафик мал (сотни посетителей/мес по Метрике; с поправкой на недоучёт ~2–3× — всё равно ранняя стадия).
3. Один канал — SEO (соцсети 3 визита/мес): концентрационный риск.
4. Lab-LCP travel-страниц ~10 c (Lighthouse mobile, GROWTH_PLAN FE-1) — если инвестор сам прогонит PSI, цифра будет плохой; field-данные и архитектурный контекст (RN-Web гидратация) объяснены в FE-1.
5. Индексация: 435 в индексе / 490 нет (снимок 12.06, ANALYTICS_AUDIT §1) — хвост чинится (фаза 3 плана).
